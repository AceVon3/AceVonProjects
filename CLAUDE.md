## Project Overview

This is a collection of single-file browser games. No build system, no dependencies, no package manager — each game is a standalone `.html` file that runs directly in a browser.

## Running the Games

Open any `.html` file directly in a browser:
- `game.html` — Neon Assault, a top-down shooter (800×600 canvas)
- `tictactoe.html` — Tic-Tac-Toe with score tracking

No server required. No compilation step.

## Architecture

Both games follow the same pattern: **all HTML, CSS, and JavaScript in a single file**.

### Neon Assault (`game.html`)

- **State machine**: `STATE` enum (`MENU`, `PLAYING`, `LEVEL_COMPLETE`, `GAME_OVER`) drives all rendering/update branching in the main loop
- **Game loop**: `requestAnimationFrame` → `gameLoop(ts)` → `update(dt)` + `render(dt)`, with `dt` capped at 50ms to prevent spiral of death
- **Entity arrays**: `bullets`, `enemies`, `particles`, `scorePopups` are filtered in-place each frame (dead entities removed via `.filter()`)
- **Enemy types**: Defined in `ENEMY_DEFS` (`grunt`, `runner`, `tank`). Level wave compositions are generated by `makeLevelDef(lvl)`, which scales enemy counts for levels 4+
- **Input**: Keyboard state tracked in `keys` object; mouse position/button tracked in `mouse` object
- **Drawing**: All rendering uses raw Canvas 2D API helpers (`drawRect`, `drawCircle`, `drawText`). No sprites — everything is drawn procedurally

### Tic-Tac-Toe (`tictactoe.html`)

- Win conditions stored as index triples in `WINS` array
- State is minimal: `board` (9-element array), `current` player, `gameOver` flag, `scores` object
- Scores persist across "New Game" resets within the same session (no localStorage)
